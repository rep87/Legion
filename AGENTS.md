# IRON FRONTIER — AGENTS.md
> 이 파일을 먼저 읽고 전체 구조를 파악한 후 작업을 시작해.

---

## 게임 한 줄 요약
버려진 행성에서 영웅이 탐험·전투하며 로봇 부품을 수집하고, 기지에서 로봇을 조립해 경로에 배치하는 타워디펜스 + 액션 RPG.

---

## 기술 스택
- HTML5 Canvas + Vanilla JavaScript (프레임워크 없음)
- 단일 페이지: index.html + js/ 폴더 분리
- 캔버스 크기: 800×600
- 그래픽: 픽셀아트, 다크 SF, 어두운 배경 + 붉은/주황 포인트

---

## 파일 구조 & 각 파일의 역할

```
index.html        진입점, 캔버스, UI 레이아웃
js/
  game.js         전역 상태(gameState), 게임 루프, 웨이브, 지역 전환  ← 모든 파일이 참조
  hero.js         영웅 이동/전투/루팅/시야 업데이트
  enemies.js      적 로봇 AI, 경로 이동, 드롭, 보스
  inventory.js    인벤토리 9칸, 조합, 판매
  robots.js       로봇 조립/배치/분해/전력/오라
  map.js          암흑시야, 경로 렌더링, 배치 슬롯, 포탑
assets/
  hero/ enemies/ robots/ towers/ ui/
```

**의존 순서: game.js → hero.js / enemies.js / map.js → inventory.js / robots.js**
- 각 에이전트는 자신이 담당한 파일만 수정한다.
- 다른 파일을 수정해야 할 경우 주석으로 TODO를 남기고 멈춘다.

## 전역 상태 (gameState) — game.js에 정의

```javascript
const gameState = {
  // 지역
  currentRegion: 'A',        // 'A' | 'B'
  regions: { A: { cleared: false }, B: { cleared: false } },

  // 웨이브
  wave: 1,                   // 현재 웨이브 (지역당 1~10)
  maxWaves: 10,
  isBossWave: false,         // 5, 10웨이브

  // 기지
  baseHP: 100,
  maxBaseHP: 100,
  power: 10,                 // 현재 가용 전력
  maxPower: 10,              // 기지 업그레이드로 증가
  usedPower: 0,              // 배치된 로봇이 소모 중인 전력

  // 경제
  gold: 100,

  // 영웅
  hero: { x: 400, y: 300, hp: 100, maxHP: 100, items: [null, null] },

  // 공유 컬렉션
  droppedItems: [],          // 바닥에 떨어진 부품/골드 아이템
  placedRobots: [],          // 배치된 아군 로봇 목록
  enemies: [],               // 현재 적 목록
  fogOfWar: [],              // 시야 오픈된 타일 좌표 목록
};
```

## 핵심 게임 메커닉 요약

### 영웅
- WASD 또는 마우스 클릭 이동
- 스페이스바 또는 좌클릭으로 근접 공격 (초반 주력)
- 바닥 아이템 마우스 클릭 → 루팅 (인벤토리 꽉 차면 경고, 루팅 불가)
- 이동 시 반경 내 암흑시야 영구 해제 → map.js에 신호 전달

### 적 로봇
- 고정 경로(Path)를 따라 이동, 기지 도달 시 baseHP 감소
- 처치 시 부품(팔/다리/머리/몸통) 1~2개 + 골드 드롭
- 등급 확률: C 60% / B 25% / A 12% / S 3%
- 드롭 아이템은 바닥에 30초간 유지 후 제거

### 부품 4종
| 부품 | 역할 | 핵심 스탯 |
|------|------|-----------|
| 팔 (무기형) | 공격 (칼/총/레이저) | 공격력, 공격속도 |
| 다리 | 이동 + 범위 지면 충격 | 이동속도, 범위 데미지 |
| 머리 | 레이저 공격 + 회복 | 공격속도, **회복속도** |
| 몸통 | 범위 오라 (패시브 버프) | 방어력, 체력, 오라 범위 |

### 등급 시스템
C → B → A → S (같은 종류 + 같은 등급 3개 조합 시 승급)

### 인벤토리
- 하단 9칸 고정 슬롯
- 꽉 차면 루팅 불가 + 경고 문구
- 3개 조합 감지 시 팝업 (강제 아님)
- 우클릭 → 판매

### 기지 전력 (Power)
- 로봇 운용 한도 = maxPower
- 로봇 조립 시 전력 소모, 분해 시 반환
- 전력 초과 시 조립 불가
- 기지 업그레이드로 maxPower 증가

### 로봇 조립 & 배치
- 조건: 팔 + 다리 + 머리 + 몸통 각 1개 + 골드
- 배치: 경로 위 지정 슬롯에만 가능 (슬롯당 로봇 3대 + 포탑 1개 제한)
- 명령: 이동 명령 1가지만
- 체력 0 → 자동 회복 (전투 불가), 완료 후 재전투
- 몸통 오라: 인접 로봇에게 패시브 버프 (등급에 비례)

### 로봇 분해
- 분해 아이템 필요
- 로봇 기지 귀환 → 분해 아이템 드래그드롭 → 부품 환원
- 현장 즉시 분해 불가 (의도적 불편함)

### 보스
- 웨이브 보스: 5·10웨이브 등장, 체력·공격력 10배
- 지역 보스: 지역 클리어 조건, 처치 시 지역 전환
- 보스는 아군 로봇에게 직접 데미지 가능
- 목표 난이도: 보스전당 로봇 2~3회 파괴·회복 반복

### 암흑시야
- 기지 주변만 초기 개방
- 영웅 이동 시 반경 내 영구 해제
- 시야 포탑 설치로 특정 구역 해제 가능

### 지역 구조
- A지역 (1~10웨이브) → 지역 보스 처치 → A 요새화
- B지역 개방 (더 강한 적, 복수 경로 가능)
- A기지에서 B지역으로 주기적 부품·로봇 지원

## UI 레이아웃

```
┌─────────────────────────────────────────────────┐
│  상단: 재화 바 (골드, 전력, 웨이브, 기지HP)         │
├──────────┬──────────────────────────┬────────────┤
│          │                          │            │
│  좌측    │      게임 캔버스           │            │
│  패널    │      (중앙 메인)           │            │
│          │                          │            │
│ - 기지   │                          │            │
│   관련   │                          │            │
│ - 로봇   │                     ┌────┴────┐       │
│   생산   │                     │ 미니맵  │       │
│          │                     │(우하단) │       │
├──────────┴─────────────────────┴────────┴───────┤
│  하단: 아이템 인벤토리 (9칸 슬롯)                   │
└─────────────────────────────────────────────────┘
```

### 각 영역 담당 파일
- 상단 재화 바: index.html + game.js
- 좌측 패널 (기지·로봇 생산): index.html + robots.js
- 게임 캔버스 (중앙): map.js
- 미니맵 (우하단): map.js
- 하단 인벤토리: index.html + inventory.js

### 미니맵 스펙
- 위치: 캔버스 우하단 오버레이 (200×150px)
- 표시 내용: 전체 맵 축소판, 영웅 위치, 적 위치, 암흑 구역, 경로
- 암흑 구역은 미니맵에서도 검게 표시
- 시야 포탑 설치 시 해당 구역 미니맵도 해제

## UI 관련 에이전트 추가 지침
